# PROMPT_03 — Proposal collateral: deck, exec summary, sample PDFs, SOW

**Prerequisite:** `PROPOSAL_CHARTER.md`, `ANTIPATTERNS.md`, and `PROMPT_01_ARCHITECTURE.md` must be complete. Can run in parallel with `PROMPT_02_DEMO.md`.

> **This system reports AUC ~0.62 ± 0.06 on synthetic data. No deliverable — UI copy, slide, PDF, code comment, README — may imply higher performance, production readiness, or validation on real hiring outcomes. Banking compliance reviewers will check. The honesty of the numbers is the product.**

## Goal

Produce the leave-behind package a banker can forward to MRM, Compliance, CISO, and TPRM without you in the room. Each artifact must survive that forwarding without misrepresenting what the system does.

## Scope ceiling

- Max new files: 9 (listed below)
- Max LOC for any single artifact: 800 lines (deck markdown), 400 lines (exec summary), PDFs templated from existing data
- If you need more, ask.

## Output location

All collateral lives in `handoffs/collateral/`. Do not put any of these at repo root.

## Per-deliverable specs

### 1. `handoffs/collateral/PITCH_DECK.md`
12 slides, markdown format suitable for `marp` or `pandoc -t pptx`. Use the structure from PROPOSAL_CHARTER.md context (cover, regulatory landscape, problem, approach diagram, differentiators, demo placeholder, compliance map, PoC scope, timeline, team placeholder, Phase 2 roadmap, CTA).

Constraints:
- Slide 6 ("Demo") is just the title "Live Demo" + a screenshot grid pointing at `src/demo/screenshots/`
- Slide 5 ("Differentiators") uses a 4-column comparison: **Us | Black-Box Vendor (generic) | ATS Built-In (generic) | In-House Build**. Do NOT name specific vendors.
- Slide 7 ("Compliance map") explicitly maps: SR 11-7 sections → our deliverable, NYC LL144 → "compatible with independent auditor (we do not self-certify)", FCRA → adverse-action notice generator, EU AI Act high-risk → conformity package
- No slide promises a specific AUC number; performance section says "validated reproducible methodology; performance bracketed on customer data during PoC"
- No animations, gradients, or "powered by AI" language
- Footer of every slide: "DEMO data. Synthetic dataset. AUC 0.62 ± 0.06."

Commit message: `docs: 12-slide proposal pitch deck (markdown)`

### 2. `handoffs/collateral/EXEC_SUMMARY.md`
Two-page (≤800 words) executive summary for the buyer who doesn't open attachments.

Structure:
- Para 1: the regulatory pressure (3 sentences)
- Para 2: our methodology (3 sentences) — name the architectural ingredients
- Para 3: what we deliver in PoC (bullet list of 5-7 items, each ≤10 words)
- Para 4: what we explicitly do NOT promise (bullet list of 4-5 items — this is a trust signal)
- Para 5: next step (one sentence — discovery workshop, 2 hours)

Tone: spare, calm, no marketing adjectives. The people reading this can sniff oversell at 100 paces.

Commit message: `docs: 2-page executive summary`

## Stop conditions

- You feel the urge to add a 10th deliverable (ask user first)
- A claim in any artifact can't be sourced from `benchmark.json` and isn't marked `[ILLUSTRATIVE]`
- The PDF generation pipeline from `PROMPT_02` isn't ready (collateral depends on it for samples 3-5)

## What NOT to do

- Do not name competitors (Eightfold, HireVue, Workday, Pymetrics, Paradox)
- Do not include real bank names, even as customers ("at [Tier-1 bank]" is fine; "at JPMorgan" is not)
- Do not generate any AUC number above 0.62 ± 0.06 anywhere
- Do not write a Phase 2 roadmap that promises specific features by specific dates
- Do not include real personal data in sample PDFs (use clearly fictional names: "Pat Sample," "Alex Demo")
- Do not write code in this prompt — collateral is markdown + generated PDFs only
